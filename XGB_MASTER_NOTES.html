01 · Project Overview

Intracranial pressure (ICP) monitoring is critical in TBI, hydrocephalus, and post-operative neurosurgical patients. The gold standard is invasive catheter insertion — a procedure with risk of infection, haemorrhage, and limited availability in low-resource settings. This project builds a non-invasive ICP classification system using optical tympanic membrane (TM) displacement sensing.

🧠 Physiological Basis
ICP transmits via the cochlear aqueduct → perilymph → round window → tympanic membrane. Elevated ICP causes measurable changes in the cardiac pulsatility amplitude, respiratory modulation, and slow-wave energy of the TM displacement signal — the same features extractable from invasive ICP waveforms (Marchbanks 1996; Skjæret-Maroni et al., Sci Reports 2021).

Core Problem Statement

Binary Classification: Given 10 seconds of physiological signals, predict whether a patient's ICP is Normal (<15 mmHg) or Abnormal (≥15 mmHg). The 15 mmHg threshold is the clinical intervention boundary used in neurocritical care.

Why Binary (not 3-class)?

⚠️ Design Decision
A 3-class model (Normal / Elevated 15–20 mmHg / Critical ≥20 mmHg) was built first. The Elevated class decision region collapsed — max predicted probability for Elevated was <3% under any input. Root cause: the 15–20 mmHg range has ambiguous waveform morphology that cannot be discriminated from 6 spectral features alone. Collapsing to binary at 15 mmHg produces an honest, well-calibrated model with a clinically meaningful decision boundary.

Window Quality Control

Each 1,250-sample window passes 4 checks before feature extraction:

def _is_valid_window(win):
    # 1. Missing data: max 30% NaN allowed
    if np.isnan(win).mean() > 0.30: return False
    clean = win[~np.isnan(win)]
    # 2. Minimum valid samples after NaN removal
    if len(clean) < WINDOW_SAMPLES * 0.5: return False
    # 3. Physiological range: ICP must be 0–50 mmHg
    if np.any(clean < 0.0) or np.any(clean > 50.0): return False
    # 4. Flatline: std < 0.01 → sensor artifact
    if clean.std() < 0.01: return False
    return True

Label Assignment

def _assign_label(icp_median):
    # Binary: median ICP of the 10-second window
    return 0 if icp_median < 15.0 else 1
    # (Original 3-class: 0=Normal, 1=Elevated 15-20, 2=Critical ≥20 — retired)
✅ Why Median, Not Mean?
ICP waveforms contain sharp spikes (cough, sneeze, position change). The median is resistant to these outliers — it reflects the sustained ICP level over the 10-second window rather than being pulled by transient artifacts.

04 · Feature Engineering

6 features are extracted from each 10-second window. They are signal-agnostic — computed identically on invasive ICP waveforms (training) and optical TM displacement (hardware inference), following the established physiological equivalence in the TM-ICP literature.

#FeatureUnitMethodRangeClinical Meaning
0cardiac_amplitudeμm Bandpass 1–2 Hz → 99th–1st percentile × 10 5–120 P1 percussion wave amplitude — pulsatility index proxy
1cardiac_frequencyHz FFT dominant peak in 0.7–2.5 Hz band 0.7–2.5 Heart rate (0.7=42 bpm, 2.5=150 bpm)
2respiratory_amplitudeμm Bandpass 0.1–0.5 Hz → 99th–1st percentile × 10 1–50 B-wave / respiratory ICP modulation — compliance indicator
3slow_wave_power— db4 wavelet level-5 approximation energy fraction 0.30–1.0 Lundberg slow-wave energy (≈ 0–1.56 Hz band); near-1 = normal
4cardiac_power— db4 wavelet level-5 detail-1 energy fraction 0–0.40 Cardiac-band wavelet energy (≈ 1.56–3.12 Hz); compliance proxy
5mean_arterial_pressuremmHg Mean of ABP channel, clamped 40–200 mmHg 40–200 CPP = MAP − ICP; most discriminative inter-patient feature

Feature Extraction Code

# cardiac_amplitude: bandpass 1-2 Hz, then peak-to-peak amplitude
cb = _bandpass(win, 1.0, 2.0)
cardiac_amp = (np.percentile(cb, 99) - np.percentile(cb, 1)) * 10.0

# cardiac_frequency: dominant FFT peak in 0.7-2.5 Hz
fb = _bandpass(win, 0.7, 2.5)
freqs = np.fft.rfftfreq(len(fb), d=1.0/TARGET_FS)
power = np.abs(np.fft.rfft(fb))**2
mask  = (freqs >= 0.7) & (freqs <= 2.5)
cardiac_freq = freqs[mask][np.argmax(power[mask])]

# slow_wave_power + cardiac_power: db4 wavelet, 5 levels
coeffs   = pywt.wavedec(x100, "db4", level=5)
energies = [np.sum(c**2) for c in coeffs]
total    = sum(energies) + 1e-10
slow_pwr = np.clip(energies[0] / total, 0, 1)   # approximation (cA5)
card_pwr = np.clip(energies[1] / total, 0, 1)   # detail-1 (cD5)

Bandpass Filter Implementation

def _bandpass(sig, low, high, order=4):
    from scipy.signal import butter, sosfiltfilt
    nyq = TARGET_FS / 2.0           # 62.5 Hz
    lo  = max(low / nyq, 1e-4)       # normalise to Nyquist
    hi  = min(high / nyq, 0.9999)
    sos = butter(order, [lo, hi], btype="bandpass", output="sos")
    return sosfiltfilt(sos, sig)  # zero-phase (no phase lag)
🔬 Why sosfiltfilt (not lfilter)?
sosfiltfilt applies the filter forward-then-backward, producing zero phase distortion. This is critical for ICP waveform analysis where the timing of the P1/P2/P3 peaks relative to each other encodes clinical information. lfilter would shift peaks in time and distort amplitude ratios.

05 · Data Split Strategy

✅ Why Patient-Level Splitting Matters
A patient with 1,000 windows contributes highly correlated samples. If those windows are split across train/test, the model memorises that patient's physiological signature — not the general pattern. Patient-level splitting ensures every window in the test set comes from a patient the model has never seen.

Dataset-Stratified Split

CHARIS and MIMIC are split independently, then merged. This prevents one dataset from dominating the test set.

Implementation (GroupShuffleSplit)

def _split_cohort(X, y, pid):
    # Step 1: 80% train+val, 20% test — by patient
    gss1 = GroupShuffleSplit(1, test_size=0.20, random_state=42)
    tv, te = next(gss1.split(X, y, groups=pid))

    # Step 2: split train+val → 87.5% train, 12.5% val → 10% of total
    gss2 = GroupShuffleSplit(1, test_size=0.125, random_state=42)
    tr, va = next(gss2.split(X[tv], y[tv], groups=pid[tv]))
    return tv[tr], tv[va], te

# Split each cohort separately
ch_tr, ch_va, ch_te = _split_cohort(X[ch], y[ch], pid[ch])  # CHARIS (pid ≤ 13)
mi_tr, mi_va, mi_te = _split_cohort(X[mi], y[mi], pid[mi])  # MIMIC (pid > 13)

# Merge — guarantees both cohorts appear in all three splits
tr = np.concatenate([ch_idx[ch_tr], mi_idx[mi_tr]])
va = np.concatenate([ch_idx[ch_va], mi_idx[mi_va]])
te = np.concatenate([ch_idx[ch_te], mi_idx[mi_te]])
⚠️ Early-Stopping Val: CHARIS-only subset
During training, the early-stopping validation set uses only CHARIS windows from the val split. CHARIS has more balanced ICP distributions and less noise — this produces a cleaner AUC signal for early stopping. The full val set (CHARIS + MIMIC) is used for threshold optimisation.

06 · XGBoost Model Architecture

Algorithm: Gradient Boosted Trees

XGBoost builds an ensemble of decision trees sequentially. Each tree corrects the errors of the previous trees by fitting the negative gradient of the loss function (logistic loss for binary classification). The final output is a weighted sum of all trees passed through a sigmoid to produce P(Abnormal).

Full Hyperparameter Set

params = {
    "objective":        "binary:logistic",   # sigmoid output → P(Abnormal)
    "eval_metric":      "auc",               # monitor AUC on val set
    "eta":              0.1,                  # learning rate — shrinks each tree
    "max_depth":        4,                    # tree depth — controls complexity
    "min_child_weight": 5,                    # min sum of hessian in leaf — prevents small splits
    "subsample":        0.8,                  # 80% of rows per tree — reduces variance
    "colsample_bytree": 0.8,                  # 80% of features per tree — like RF
    "scale_pos_weight": spw,                  # = n_normal / n_abnormal (handles imbalance)
    "lambda":           1.0,                  # L2 regularisation on leaf weights
    "alpha":            0.1,                  # L1 regularisation — sparsity
    "seed":             42,                   # reproducibility
    "tree_method":      "hist",              # histogram-based — fast on large datasets
}

bst = xgb.train(
    params, dtrain,
    num_boost_round=500,            # max 500 trees
    evals=[(dtrain,"train"), (dval,"val")],
    early_stopping_rounds=30,       # stop if val AUC doesn't improve for 30 rounds

)

07 · Class Imbalance Handling

MIMIC-III contains predominantly Normal ICP windows (general ICU patients don't all have elevated ICP). Without correction, the model would learn to predict "Normal" for everything and achieve high accuracy while being clinically useless.

scale_pos_weight

n0 = (y_tr == 0).sum()   # count Normal windows in training set
n1 = (y_tr == 1).sum()   # count Abnormal windows
spw = n0 / n1             # e.g. 0.48 if 68% Normal, 32% Abnormal
# XGBoost multiplies the gradient of positive class by spw
# → model penalises missing Abnormal cases more than missing Normal
📐 Mathematical Effect of scale_pos_weight
In binary logistic loss: L = -[y·log(p) + (1-y)·log(1-p)]
With scale_pos_weight = w: L = -[w·y·log(p) + (1-y)·log(1-p)]
This is equivalent to oversampling the positive class by factor w, but computationally more efficient — no actual resampling occurs.

Why Not SMOTE or Oversampling?

SMOTE creates synthetic windows by interpolating between real windows. For physiological time-series features, synthetic ICP windows may not correspond to any real physiological state — potentially teaching the model nonsensical patterns. scale_pos_weight modifies the loss directly without data fabrication.

08 · Threshold Selection

XGBoost outputs P(Abnormal) ∈ [0, 1]. The threshold τ converts this to a binary decision. Default τ=0.5 is suboptimal for clinical use — we want high recall (don't miss abnormal cases).

Youden's J Statistic

# Compute ROC curve on validation set
fpr, tpr, thresholds = roc_curve(y_va, val_probs)

# Youden's J = TPR - FPR = Sensitivity + Specificity - 1
# Maximising J gives the threshold that best balances sensitivity and specificity
j_idx = np.argmax(tpr - fpr)
threshold_youden = thresholds[j_idx]

# Clinical override: also find best F1 where Recall ≥ 0.75
# (Missing abnormal ICP is more dangerous than false alarms)
for t in np.linspace(0.1, 0.9, 161):
    preds = (val_probs >= t).astype(int)
    f = f1_score(y_va, preds)
    r = recall_score(y_va, preds)
    if r >= 0.75 and f > best_f1:
        best_f1, threshold = f, t   # selected threshold
⚠️ Important: Threshold on Val, Applied to Test
The threshold is found on the validation set only. It is then applied to the test set — the test set never influences any decision during training or threshold selection. This preserves test set integrity.

Patient-Level Bootstrap Confidence Intervals

def bootstrap_ci(y_true, y_pred, y_prob, pid, n_boot=1000):
    unique_pids = np.unique(pid)
    for _ in range(n_boot):
        # Resample PATIENTS with replacement (not windows)
        sampled = rng.choice(unique_pids, size=len(unique_pids), replace=True)
        # Pool all windows belonging to sampled patients
        idx = np.concatenate([np.where(pid==p)[0] for p in sampled])
        # Compute metrics on pooled windows
        boot_f1.append(f1_score(y_true[idx], y_pred[idx]))
        boot_auc.append(roc_auc_score(y_true[idx], y_prob[idx]))
    # 95% CI = [2.5th percentile, 97.5th percentile] across 1000 iterations
🔑 Why Patient-Level Bootstrap?
Windows from the same patient are correlated (same physiology). If you resample individual windows, the same patient appears in nearly every bootstrap sample → CIs are artificially narrow by 3–5×. Patient-level resampling respects the true unit of analysis and produces honest, wider confidence intervals.

Expected Calibration Error (ECE)

def _ece(y_true, probs, n_bins=10):
    bin_edges = np.linspace(0.0, 1.0, n_bins + 1)
    ece = 0.0
    for lo, hi in zip(bin_edges[:-1], bin_edges[1:]):
        mask = (probs >= lo) & (probs < hi)
        if not mask.any(): continue
        # In a perfect model: mean predicted prob = fraction of positives in bin
        ece += (mask.sum() / n) * abs(probs[mask].mean() - y_true[mask].mean())
    return ece  # ECE = 0 → perfect calibration

RF also builds decision trees with bagging — fundamentally similar to XGBoost. 13 · SHAP Explainability

SHAP (SHapley Additive exPlanations) decomposes each individual prediction into per-feature contributions. For a window predicted as Abnormal, SHAP explains which features pushed the prediction above the threshold and by how much.

# Sum = log-odds of prediction; sigmoid(sum) = P(Abnormal)

# Negative SHAP → feature pushed toward Normal
🏥 Clinical Use of SHAP
Instead of a black-box "Abnormal: 87%" output, the clinician sees: "High MAP (+0.42), elevated cardiac amplitude (+0.31) drove this Abnormal prediction. Slow wave power was normal (−0.08)." This enables clinical trust and error checking — if the top SHAP feature is a known artifact for that patient, the clinician can discount the alert.

16 · Hard Q&A

HARD Why XGBoost and not a neural network for this task?

Three reasons rooted in the data and problem structure:

  1. Tabular features, not raw waveforms: We extract 6 engineered features per window, not raw 1,250-sample waveforms. XGBoost is empirically stronger than MLPs on tabular data with <100 features (Grinsztajn et al., NeurIPS 2022). The features already encode the temporal structure.
  2. Interpretability requirement: Clinical decision support systems need explainability. XGBoost provides exact SHAP values via tree SHAP (Lundberg & Lee, NeurIPS 2017). Neural networks require approximate SHAP which is computationally expensive and less reliable for tree-structured decision boundaries.
  3. Data size efficiency: XGBoost converges reliably with 100 patients. Deep networks for tabular data tend to overfit or underfit with <1M samples unless heavily regularised.

The LSTM is used separately for forecasting (temporal sequence → future state), where the sequential structure of raw sequences matters. Classification of individual windows → XGBoost. Trend forecasting over sequences → LSTM.

HARD How does GroupShuffleSplit differ from StratifiedKFold, and why does it matter here?

StratifiedKFold stratifies by class distribution but operates on individual samples — it can put window 500 from Patient A in training and window 501 from Patient A in the test set. For time-series physiological data, this causes data leakage: the model sees the patient's physiological baseline in training and is then tested on data from the same baseline.

GroupShuffleSplit with groups=patient_id enforces that all windows from a given patient go to either train OR test — never both. The "shuffle" means groups are randomly assigned to splits. The "group" guarantee means no intra-patient leakage.

In our case, a single patient can have 5,000+ windows. Without group-based splitting, the model would achieve inflated metrics by memorising per-patient physiological signatures rather than learning generalizable ICP patterns. Our cross-dataset eval (CHARIS → MIMIC) further validates that the patterns generalise to unseen patients from a different hospital.

HARD Why was the threshold not set at 0.5? What is Youden's J and when does it fail?

P=0.5 is the optimal threshold only when the cost of a False Positive equals the cost of a False Negative AND the class distribution is balanced. Both conditions fail here:

  • Asymmetric costs: A False Negative (missed Abnormal ICP) means a deteriorating patient goes unmonitored. A False Positive (false alarm) wastes clinical time but doesn't harm the patient. FN is more costly.
  • Class imbalance: More Normal windows → model is biased toward Normal → default 0.5 threshold misses abnormal cases.

Youden's J = max(TPR - FPR) is equivalent to maximising Sensitivity + Specificity - 1. It finds the threshold that best separates the two classes in ROC space.

When Youden's J fails: If the clinical cost ratio is extreme (e.g., FN is 100× worse than FP), you should use the threshold that achieves a minimum required recall (sensitivity), not the one that maximises balanced accuracy. That is why we add the constraint: find the best F1 threshold WHERE recall ≥ 0.75. Youden's J is used only as a fallback if no threshold achieves recall ≥ 0.75.

HARD What is Expected Calibration Error and why is a well-calibrated model important clinically?

A model is calibrated if P(Abnormal) = 0.80 means that, among all windows where the model outputs 0.80, approximately 80% are truly Abnormal. A miscalibrated model might output 0.80 for windows where only 60% are Abnormal — the probability is meaningless.

ECE formula: Divide [0,1] into n bins. For each bin, compute |mean predicted probability − fraction of true positives|, weighted by how many samples fall in that bin. ECE = 0 is perfect; ECE = 0.1 means predictions are off by 10% on average.

Clinical importance: Physicians don't just need a binary alert — they need to know how confident the model is to triage multiple patients. If the model says P=0.95 for Patient A and P=0.62 for Patient B, a well-calibrated model means "check Patient A first." If the model is miscalibrated, the probability is untrustworthy for prioritisation.

In our pipeline, we measured ECE before and after isotonic calibration. The current model uses raw probabilities (calibration was found to squish probabilities and reduce recall — raw probs were better calibrated than the calibrated version on this dataset).

HARD Why does scale_pos_weight = n_normal / n_abnormal specifically, not some other ratio?

XGBoost's binary logistic loss gradient for a positive sample (y=1) is: g = pred - y = p - 1. For a negative sample: g = pred - y = p - 0 = p.

scale_pos_weight = w multiplies the gradient AND hessian of positive samples by w. This is mathematically equivalent to replicating each positive sample w times. Setting w = n_negative / n_positive makes the total weighted contribution of both classes equal — the model trains as if the dataset were perfectly balanced.

Why not higher? If you set w = 3× the ratio, you overcorrect: the model aggressively predicts Abnormal for everything, achieving high recall but collapsing precision to near the class prior. Why not lower? Under-weighting underrepresented class → the model ignores Abnormal windows. The ratio = exact balance point.

HARD What is the difference between the db4 wavelet energy ratio features and a simple FFT power feature?

Both measure energy in frequency bands, but they differ in time-frequency resolution:

FFT: Global frequency resolution. Perfect frequency resolution, zero time resolution. The FFT of a 10-second window tells you exactly what frequencies are present, but not when they occurred. For stationary signals this is fine.

Wavelet (db4, level 5): Multi-resolution. At low frequencies (slow-wave band), uses long time windows (good frequency resolution). At high frequencies (cardiac band), uses short time windows (good time resolution). This matches the nature of ICP signals: slow waves are stationary over 10 seconds; cardiac pulses are transient spikes. The db4 (Daubechies 4) wavelet has a shape similar to the ICP cardiac pulse — this makes it particularly sensitive to ICP-relevant pulsatility patterns.

In practice, the wavelet energy fraction is more robust to non-stationarities (patient movement, cough artifacts) than FFT power because the wavelet's compact support means transient artifacts don't contaminate the entire spectrum.

HARD How does the cross-dataset evaluation (train CHARIS → test MIMIC) prove generalisability, and what are its limits?

What it proves: The 6 spectral/wavelet ICP features encode physiological patterns that are consistent across two different ICU populations (TBI-specific CHARIS vs. general-ICU MIMIC), different hospitals, different monitoring equipment, and different time periods (CHARIS is older data). If the model performed well only on CHARIS-held-out but failed on MIMIC, it would mean it memorised TBI-specific ICP morphology rather than general ICP physiology.

Limits:

  1. Domain shift direction: We train on 13 patients, test on 87. MIMIC has far more data and diversity. We're testing generalisation from a small specialist cohort to a large general cohort — a harder problem than the reverse.
  2. Both are invasive catheter datasets: The cross-dataset eval validates generalisation across hospitals, not across sensor modalities. The leap from catheter waveforms to optical TM displacement is validated by literature (Marchbanks, Skjæret-Maroni), not by this experiment.
  3. MIMIC population heterogeneity: MIMIC includes post-op, septic, and cardiac patients whose ICP physiology may differ from TBI. The model must learn ICP-specific patterns, not disease-specific ones.
HARD Why is the bootstrap CI patient-level instead of window-level, and how much does it change the result?

The problem with window-level bootstrap: With 5,000 windows from Patient A and 3,000 from Patient B in the test set (8 patients total), each bootstrap sample (resample with replacement from 8,000 windows) will contain Patient A's windows ~5,000/8,000 ≈ 62.5% of the time. Patient A dominates every single bootstrap sample. The variance of the metric across bootstrap iterations reflects Patient A's within-patient variability, not the variance from having different patients.

Patient-level bootstrap: Resample the 8 patients with replacement. Some bootstraps may have Patient A twice and Patient F not at all. The variance across iterations now reflects what would happen if we had different patients in our test set — the true source of uncertainty for a clinical deployment.

Magnitude of difference: Typically, window-level CIs are 3–5× narrower than patient-level CIs on medical time-series data. A window-level 95% CI of [0.87, 0.89] might become [0.82, 0.92] with patient-level resampling. The point estimate doesn't change — only the uncertainty quantification.

HARD What is McNemar's test and why use it instead of a paired t-test on AUC values?

McNemar's test operates on the 2×2 table of concordant/discordant predictions between two models on the same test set:

  • b = windows where Model A is correct, Model B is wrong
  • c = windows where Model A is wrong, Model B is correct

Under H₀ (both models make the same errors): b ≈ c. Test statistic: χ² = (|b−c| − 1)² / (b+c), df=1.

Why not a t-test on AUC? AUC is computed from the ROC curve — it's a single scalar per model, not a distribution. A t-test requires multiple independent samples of AUC (e.g., from k-fold CV). McNemar's test uses the per-window predictions directly on the same test set — it's more statistically powerful because it accounts for the correlation between predictions (both models see the same windows). DeLong's test is the AUC-specific equivalent, also valid, but McNemar on binary predictions is simpler to explain and interpret.

HARD The cardiac_amplitude feature multiplies by 10 — why, and could this create distribution shift?

The multiplication by 10 converts the raw peak-to-peak bandpass amplitude from mmHg (or arbitrary units) to approximate micrometers (μm) — the unit used in the TM displacement literature for reporting pulsation amplitude. ICP P1 wave amplitudes are typically reported in the 10–80 μm range for TM displacement measurements. The ×10 scaling maps the raw ICP-derived amplitude into the same numerical range, making the FEATURE_RANGES (5–120 μm) physically interpretable.

Distribution shift risk: None, because the scaling is a global constant applied identically to all windows during both training AND inference. The XGBoost model learns thresholds on the scaled values. As long as the scaling is applied consistently (enforced via config.py and the backend's validation), there is no shift. The risk would only appear if the hardware applied a different scaling — which is why config.py is the single source of truth for all feature ranges and the backend validates incoming features against those ranges before inference.

HARD Why use `hist` tree_method instead of `exact`? What is the tradeoff?

Exact method: For each feature, sorts all training samples and evaluates every possible split point. Time complexity O(n·d·log n) per boosting round where n = samples, d = features. With 448K windows × 6 features × 500 rounds, this would take hours.

Hist method (approximate): Pre-bins each feature into ~256 bins using a histogram. Split finding then operates on 256 candidate splits per feature instead of n. Time complexity O(n·d) for the initial histogram construction, then O(256·d) per round. 10–50× faster on large datasets.

Quality tradeoff: Hist can miss the theoretically optimal split point if it falls between two bin edges. In practice, with 256 bins over the continuous feature range, the approximation error is negligible — the empirical AUC difference between exact and hist is typically <0.001 on datasets of this size. The XGBoost documentation recommends hist for any dataset with >10K samples, which is our case at 448K.

PROF If a professor asks: "Is ICP reliably predictable from only 6 features?" — how do you answer?

Direct answer: Reliably classifiable (binary, above/below 15 mmHg) — yes, at AUC 0.949 across 100 patients from two independent datasets. Precisely predictable (exact ICP in mmHg) — no, and we don't attempt that.

The deeper argument: The 6 features encode the four physiological ICP indices described in the literature — pulsatility index (cardiac_amplitude), compliance ratio (cardiac_power/slow_wave_power), cerebral perfusion pressure proxy (MAP), and respiratory ICP modulation (respiratory_amplitude). These are the same parameters clinical physicians assess when interpreting ICP waveform morphology. XGBoost learns the non-linear combinations of these clinically validated indices that separate Normal from Abnormal — it's not learning arbitrary correlations.

The honest limitation: 6 features cannot capture all sources of ICP variation. Patient-specific anatomy (CSF space volume, brain compliance), medication effects (mannitol, sedation), and sensor placement variability all affect ICP without being encoded in these features. The 18.1% recall miss rate (1 − 0.819) likely represents these unobservable confounders rather than model failure.

PROF What would you do differently if you had 6 months and hospital data access?
  1. Collect simultaneous TM + invasive ICP data (N=30 patients). Even 5 patients would validate the sensor modality transfer. This is the single highest-impact addition.
  2. Add patient-specific features: Age, GCS score, diagnosis (TBI vs hydrocephalus vs tumour), current medication (mannitol dose, sedation level). These are strong confounders.
  3. Bayesian hyperparameter search (Optuna) instead of hand-tuned params. CV-guided search would likely find a better depth/lr/regularisation combination.
  4. Temporal XGBoost: Instead of treating each 10-second window independently, use a sliding window of XGBoost predictions as input to the next prediction — capturing ICP trends without the full LSTM complexity.
  5. Prospective deployment study: Run the monitor alongside a real ICU for 30 days, comparing alerts to clinical team decisions. This produces real-world sensitivity/specificity and identifies failure modes.

17 · Glossary

ICP
Intracranial Pressure — pressure inside the skull. Normal: 5–15 mmHg. Abnormal: ≥15 mmHg.
MAP
Mean Arterial Pressure = (SBP + 2×DBP) / 3. CPP = MAP − ICP.
CPP
Cerebral Perfusion Pressure. Below 60 mmHg → ischemia risk. Makes MAP the most discriminative feature.
TM Displacement
Tympanic membrane movement caused by ICP-driven perilymph pressure changes. The non-invasive signal used by our sensor.
AUC-ROC
Area Under ROC Curve. 0.5 = random; 1.0 = perfect. Threshold-independent discrimination measure.
ECE
Expected Calibration Error. Measures how well predicted probabilities match actual outcome frequencies.
SHAP
SHapley Additive exPlanations. Decomposes each prediction into per-feature contributions. Exact for tree models.
GroupShuffleSplit
scikit-learn splitter that ensures all samples from a group (patient) appear in only one split partition.
scale_pos_weight
XGBoost parameter = n_negative / n_positive. Reweights loss so minority class gets equal total gradient contribution.
db4 Wavelet
Daubechies-4 wavelet. 4 vanishing moments. Shape similar to ICP cardiac pulse — optimal for ICP pulsatility analysis.
sosfiltfilt
Second-order sections (SOS) filter applied forward+backward. Produces zero phase distortion. Critical for waveform timing analysis.
McNemar's Test
Statistical test for comparing two classifiers on the same test set. Uses discordant prediction pairs. χ² distribution, df=1.
Youden's J
J = Sensitivity + Specificity − 1 = TPR − FPR. Maximised at the optimal ROC threshold.
Lundberg Waves
Slow ICP oscillations: A-waves (0.5–2 min, plateau ≥50 mmHg, pathological), B-waves (0.5–2 min, smaller). Captured by slow_wave_power.
CHARIS
PhysioNet TBI ICU dataset. 13 patients, simultaneous ICP+ABP. Used as primary training cohort.
MIMIC-III WDB
PhysioNet general ICU waveform database. ~7000 records, ~87 with ICP channel. Streamed via wfdb.
hist tree_method
XGBoost approximate split-finding using feature histograms. O(n·d) vs O(n·d·log n). Required for >100K samples.
Bootstrap CI
95% confidence interval from 1000 resampled patient populations. Patient-level resampling required for clustered medical data.
